Combining Selectors

Combining selectors is a more precise way to target dumb elements. Here we will look some the most common ways of combining selectors.

  1. The first one is a "p" selector which has a "dot" big class selector:

    This expresses that we want to target every "p" element that has an attribute class equal to big. Note: there are no spaces between "p" and "dot big".

    Notice here that only the "p" elements that have a class equal to big will be affected while the "div" element is unaffected.

  2. The second type of selector is a child selector. The sytanx is a selector followed by another selector.

    This way of combining is read from right to left. Meaning that the "p" element is a direct child of an "article" element.

    The only "p" element that is affected is a direct child to "article" as opposed to a regular "p" which will be unaffected. Also note, "p" not a direct child of "article" will be unaffected. Since it contained in the "div" it is not a direct child of "div".
  3. Another type of selector is a descendent selector. The syntax is selector space selector. It targets every "p" that is inside (at any level) of article.

    Even if "p" is not a direct child of "article," but as long one of its parents in its ancestry is an "article" the rule will apply.

    The both "p" elements in "article" will turn blue, despite the second being part of "div" and not a direct child.

  4. Selector combinations are not limited to element selectors.

    The first combination, ".colored p," says that every p that is inside, at any level, an element with a class equal to coloured will turn blue. The second one says taht every element with class = ".coloured" that is a direct child of "article" will turn blue.

Conflict Resolution

Your style placement affects how reusable the styles are and it also affects which style declarations override other style declarations.

Cascading is a fundamental feature of CSS. It's an algorithm defining how to combine properties values originating from different sources. As the name itself suggests, cascading style sheets, in other words the cascade algorithm, is at the core of understanding and using CSS. The cascade combine the importance, origin, specificity and source order of the applicable style declarations to determine exactly which declaration should be applied to any given element. And if there's a conflict, how to resolve that conflict. In other words, how to tell which CSS rule wins.

To understand how cascading works it is crucial to understand these four concepts:

  1. Origin or Origin Precendence

    • When two declarations are in conflict, in other words they specify the same property for the same target, origin precedence rule kicks in, and it's a very simple rule. And the rule is, the last declaration wins.
    • Now, when trying to figure out what the last declaration is, you have to remember that HTML is processed sequentially. That means top to bottom. So as you see the declarations happen, the lower on the page they are, the more precedence they have. And also for precedence, think of external CSS as declared at the spot where it's linked to. So usually it's declared in the head somewhere, but there could be other styles that I declared in the head. And to figure out which one was declared last, imagine that the entire contents of an external CSS file were cut and pasted straight into the head portion, where that external CSS is declared.

  2. Merge

    • When different CSS declarations do not conflict, that is, they still target the same element, but the CSS properties with which they target that element are different, there's even a simpler rule. And that is that declarations merge. So a declaration for, for example, font size, and a declaration for color, since they're two different properties, when they're targeted to the same element, even if they're targeted from two different origins, they will merge into one. And the element will get both the font size and the color.

  3. Inheritance

    • The basic idea is that you have the document object model tree. And if you specify some CSS property on some element, all the children and grandchildren and so on and so on of that element will also inherit that property without you having to specify the property for each and every element. So for example, if I specify one property on the body tag, every element that is a child of a body tag, or even a child of a child of a body tag and so on, will inherit that property.

  4. Specificity

    • Specificity has a pretty simple rule,]and that the most specific selector combination wins.

    • You think of the selectors as keeping a score, the selectors with highest score wins. The selector with highest score would be considered the most specific.
    • It's easier to calculate the score if you arrange the types of things that affect the score from left to right, with the left being the highest value of specificity. Then simply take your CSS rule and fill in the number of times you see a particular type of selector being used in it's proper location. The number that's created is your final score. On the screen, the most specific targeting that exists in CSS doesn't actually use any selectors at all and that is the style attribute on an element. The ID, Class, pseudo-class, attribute, and # of elements.

    • In this instance we are using a style attribute inside the h2 element. This produces a score of 1000 and it is the highest score you can get. The rest of the selectors get zeros.

    • In this instance there is no definition inside the style atribute which is zero, no id which is zero, no class which is zero, and it defines two elements "div" and "p" which is equal to two.

    • With these two cases, we try to figure which selector combinations would win.

    • The combination on the left would win since it has a higher score.

    • You can override all these rules with "! important". Avoid using important unless you absolutely have to use it.

Positioning

The position property in CSS has four legal values (in addition to the ubiquitous inherit):

  • static
  • relative
  • absolute
  • fixed

These two values static and relative are closely related. Absolute and fixed are also closely related. These schemes allow you to specify precise offsets to move the target element to the different part of the page origin.

Rectangles

In HTML when an element is rendered on a computer screen or printed on paper, those elements generate rectangular boxes. Just as the set of HTML elements is divided into block-level elements and inline elements, boxes in CSS are essentially either block boxes or inline boxes.

p and div generate bock boxes, while inline elements such as strong and span generate inline boxes. We can control the type of box that is generated using the display.

Static positioning

Boxes with position: static are not really "positioned" at all in the CSS sense. They appear in way they are already ordered in mark up and take up as much space as they need. This is default behavior.

There are significant difference in how block boxes and in-line boxes are laid out.

Block box layout

Unless specified, block boxes are laid out vertically from top to bottom in the order they occur in the markup.

  • Each box is normally as wide as the document(the body element), even if we make them narrower they will still be laid out below one another.
  • The vertical distance is controlled by the margin-bottom and margin-top properties.
  • Dimensions can be specified using width and height
  • The default value for width and height is auto. This means a block box will be wide as its parent

Inline box layout

Inline bozes appear horizontally as they are ordered in markup.

  • Using the direction property, the inline boxes will either be laid out from left to right (direction:ltr) or from right to left (direction:rtl)
  • The height of the line boxes can be changed using line-height
  • It is important to note, for inline boxes you can not change or specify an dimensions. You can only specify horizontal margins, but not vertical margins.
  • The inline box, if necessary, will be split into several inline boxes, distributed over two or more inline boxes. When this split occurs, any horizontal margins and padding, and any vertical borders, will apply before the first box and after the last box.

To vertically align inline boxes within the the encompassing line box, you can use the vertical-align propery.

  • The default value is baseline. Meaning the boxes are aligned so their text baselines line up.

Relative Positioning

Relative positioning is more closely related to static "positioning" compared to absolute and fixed positioning.

Static position is a way of saying 'normal document flow'. This is the default for all, expect HTML. Positioning offsets set to static are ignored.

An element with position:relative is laid out like any static element; block-level or inline. However, when generated the box can be arranged according to:

  • top, bottom, left, and right (offset) properties.

The element is NOT taken out of normal document flow

  • Even if moved, its original spot is preserved
  • As far as other elements are concerned, the element 'takes up space' in the original document flow.

The properties specify the edge that the movement is put up to as opposed to direction of movement. The top property shifts the box relative to it's top edge, the left relative to it's left edge, etc.

Absolute positioning is based on the idea that all offsets (top,bottom,left,right) are relative to the position of the nearest ancestor which has positioning set on it, other than static

  • Element is taken out of normal document flow, if it is set to absolute
