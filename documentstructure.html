Anatomy, Structure, Content, Semantic elements of HTML

In lectures 3 and 4, we learned about the anatomy and document structure of html.

Here is an example of an html line, "< p > '.

In the following, I will explain what each character means.

"< p >" or " < h1 >" or "< br >" are what is called element names .

"< p >" is an opening tag while "< /p > "is a closing tag . Some elements do not have a closing tag "< br >" and " < hr >" are good example of this.

This is an example of an attribute within a tag, "< p id = "myId" > < /p >". An attribute can only be specified in the opening tag.

This is an example of a no content tag "< p >< /p>". This tag is correct, while this tag "< p \>" is incorrect

We also looked as some html tags such < html >, < head>, < meta >, and < title >. A key reminder is that html is rendered sequentially meaning top to bottom.
